Lecture 10 :  Inter-process Communication

什么是IPC？

谁需要通讯？

  • 同一计算机的多个程序间
  • 同一计算机的多个进程间
  • 网络内的不同计算机

如何实现通讯？

  • Signals
  • Shared Memory
  • Pipes
  • Sockets
  • Message Queue

为什么需要IPC？

Threads 线程

线程间共享内存空间使得所有线程均在同一进程内完成通讯。线程间的通讯不需要跨进程。

Processes 进程

每个进程都有自己的一部分独立的系统资源，彼此是隔离的。为了能使不同的进程互相访问资源并进行协调工作，才有了进程间通信。通常情况下不同进程间不会设置共享内存空间。 为了使得数据完成交换的请求， 操作系统需要提供一些机制用以完成通讯。

进程间通讯机制表

  • Client connects to a socket file (in the filesystem) to reach the server
  • Server receives and accept the connection
  • OS then open a file descriptor to allow both client and server to communicate with each other (acting like a socket)
  • This socket connection can send data in both directions, using two circular buffers (one send-to and one send-from)
方法 描述
Signals
信号		 Purpose: to inform a process of an asynchronous event
Shared Memory
共享内存
Purpose: to share the same virtual memory segment among multiple process
  • A memory segment is shared among multiple processes
  • This shared segment size must be set manually so that OS can utilize a mutual exclusion mechanism to enforce concurrency protection (allowing only one process to modify its data at a time – integrity)
  • The mutual exclusion resources can be stored in shared memory segment
Pipes
管道
Purpose: to feed one process with another one's output by connecting two processes

Unnamed Pipes
  • Can only be shared between related processes
  • Created using the pipe system call.

Named Pipes
  • A special kind of pipe with a filename in the filesystem.
  • Can be used between unrelated processes.
  • One process then opens the ‘file’ read-only, while the other process opens the same file ‘write-only’.
Sockets
接口		 Purpose: to communicate between multiple clients (multiple Chrome window) and one server (display)
Message Queues
消息队列		 Purpose: to send fixed-length messages between processes
